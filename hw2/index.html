Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

Casteljiau’s algorithm is a recursive method used to evaluate and construct Bezier curves. The algorithm interpolates between control points and repeatedly interpolates on the results. This recursive process continues until there is only one point left. That point on the bezier curve corresponds to the position of the curve at a given parameter value t. Approach/Implementation: For this part, I implemented one step of Casteljiau’s algorithm to evaluate Bezier curves at some given parameter value t. The function BezierCurve::evaluateStep takes a vector of 2D control points and calculates a new set of intermediate control points by linearly interpolating between each adjacent point. My implementation iterated through the input vector and computed the interpolated point for each neighboring pair; the results were stored in a new vector. The function returns this new vector. Each call to the function performs one step of interpolation, reducing the number of points by one. The last call to the function eventually gives a single point on the Bezier curve at parameter t. Issues: One issue I encountered was my program erroring because I was iterating through the entire input vector. This was wrong because I was indexing outside of the actual vector when the loop got to the last element in the vector. I fixed this by stopping the loop 1 element before the last element of the vector.

Part 2: Bezier surfaces with separable 1D de Casteljau

Casteljiau’s algorithm extends from Bezier curves to Bezier surfaces by applying the same 1D interpolation in both directions. For a Bezier surface, the control points are arranged in a 2D grid rather than just a line. The goal is to evaluate a point on the surface at two parameters (u, v) rather than just 1 parameter t. You treat each row of control points as its own Bezier curve problem and apply Casteljiau with parameter u. This produces a set of new intermediate points which forms another 1D Bezier curve which is evaluated again using Casteljiau’s with parameter value v. The result is a single point that lies on the Bezier surface. In my implementation, I reused the same interpolation logic from part 1 on a Vector3D object in BezierPatch::evaluateStep. BezierPatch::evaluate1D repeatedly calls evaluateStep to get the last remaining point. Finally, BezierPatch::evaluate(u, v) uses evaluate1D to find the position on the Bezier surface at (u, v).

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

The goal of part 3 is to achieve smooth shading across a given mesh. Prior to part 3, each triangle was shaded separately which resulted in a sort of blocky and unrealistic look. To achieve smooth shading, we compute a normal vector at each vertex through averaging the normals of all the triangles that touch that specific vertex. Vertex::normal, the function implemented in this part, returns an approximate unit normal given some vertex. To implement this, I used the half-edge structure to “walk” around all the faces connected to the given vertex. For each triangle touching the vertex, I compute the face normal by the cross product of two edges. I then weight this calculated normal by the triangle’s area, sum all these normals together, and normalize the summed result. The final result is the return value of Vertex::normal. I ran into some issues with treating iterators like pointers. For example, I tried to do boolean checks with iterators, like you can with booleands, which did not work as iterators aren’t nullable. As a result, I had to remove any boolean checks with iterators and simply use the iterator’s methods.

Part 4: Edge flip

My approach for implementing edge flips is as follows. Given some valid edge - cannot flip boundary edges - I identified the 4 vertices, 2 faces, and 6 halfedges belonging to the two adjacent triangles connected by the given edge. I then flipped the edge by rewiring the edge to the 2 other vertices that were not connected by the original given edge. Issues: An issue I ran into was only updating the flipped edge and ignoring all surrounding connectivity. Because of this, the other halfedges belonging to the two adjacent triangles still referenced the old configuration. This led to program crashes when the program attempted to follow incorrect pointers as well as incorrect traversals. To fix this, I had to completely revamp the local connectivities. All 6 of the halfedges and their next, vertex, edge, and face pointers had to reflect the new configuration of the triangles after the edge flip.
